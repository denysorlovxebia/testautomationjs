# GitHub Secrets Configuration Guide

## Додавання секретів до GitHub

Для запуску тестів у GitHub Actions вам потрібно додати наступні секрети до вашого репозиторію:

### Кроки:

1. **Перейдіть до налаштувань репозиторію:**
   - Відкрийте ваш репозиторій на GitHub
   - Натисніть на вкладку `Settings`

2. **Знайдіть Secrets and variables:**
   - У лівому меню натисніть `Secrets and variables` → `Actions`

3. **Додайте кожен секрет:**
   - Натисніть кнопку `New repository secret`
   - Для кожного секрету введіть назву та значення:

### Необхідні секрети:

| Назва | Значення | Приклад |
|-------|----------|---------|
| `BASE_URL` | URL вашого додатку | `https://practicesoftwaretesting.com` |
| `USER_EMAIL` | Email для тестування | `customer@practicesoftwaretesting.com` |
| `USER_NAME` | Ім'я користувача | `customer@practicesoftwaretesting.com` |
| `USER_PASSWORD` | Пароль користувача | `welcome01` |

### Збереження секретів:

Після того, як ви введете кожен секрет, натисніть кнопку `Add secret` для його збереження.

## Перевірка

Після додавання секретів:

1. Створіть або змініть будь-який файл у репозиторії
2. Натисніть `git push`
3. Перейдіть на вкладку `Actions` у вашому репозиторії
4. Переглядайте статус запущених workflow файлів

## Workflow файли

Ваш проект містить три workflow файли:

- **playwright.yml** - запускається на всіх push та pull request
- **smoke.yml** - запускається щоденно о 8:00 AM та на push до main
- **regression.yml** - запускається щоденно о 20:00 та на push до main/develop

## Усунення неполадок

Якщо тести не запускаються:

1. Перевірте, чи всі секрети додані правильно
2. Переглядайте logs у вкладці `Actions`
3. Переконайтеся, що значення секретів співпадають з вашими локальними `.env` значеннями
